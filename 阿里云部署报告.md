## 五、代码层面改动

### 5.1 `src/main/resources/application.yaml`

将硬编码的 IP 和密码替换为 Spring Boot 占位符，支持通过 `application-prod.yml` 覆盖：

```yaml
spring:
  datasource:
    url: jdbc:mysql://${MYSQL_HOST:localhost}:3306/hmdp?...
    username: ${MYSQL_USER:root}
    password: ${MYSQL_PASSWORD:abc123}
  redis:
    host: ${REDIS_HOST:192.168.137.128}
    password: ${REDIS_PASSWORD:123321}
  ai:
    openai:
      api-key: ${DASHSCOPE_API_KEY:...}
    vectorstore:
      milvus:
        client:
          host: ${MILVUS_HOST:192.168.137.128}
```

### 5.2 `RedissonConfig.java`

将硬编码的 `redis://192.168.137.128:6379` + 密码 `123321` 改为从 Spring 配置读取：

```java
@Value("${spring.redis.host}")   private String redisHost;
@Value("${spring.redis.port}")   private int redisPort;
@Value("${spring.redis.password}") private String redisPassword;
```

### 5.3 `VoucherOrderServiceImpl.java`

在 `@PostConstruct init()` 方法中添加 Redis Stream 消费组自动创建（`XGROUP CREATE ... MKSTREAM`），解决新 Redis 实例缺少 `stream.orders` 和消费组 `g1` 的问题。

### 5.4 `pom.xml`

MySQL 驱动从 `mysql:mysql-connector-java:8.0.30` 升级为 `com.mysql:mysql-connector-j:8.0.33`，修复连接器 URL 兼容性 bug。

### 5.5 新增文件

| 文件 | 用途 |
|------|------|
| `application-prod.yml` | 生产环境配置（数据库、Redis、AI API、Milvus） |
| `docker-compose.yml` | 服务器依赖服务编排 |
| `start.sh` | 应用启动脚本（激活 prod profile） |
| `hmdp-nginx.conf` | Nginx 反向代理配置 |

## 六、部署操作步骤

### 初次部署

```bash
# ===== 本地操作 =====
# 1. Maven 打包（IDEA 右侧 Maven → Lifecycle → package）
cd D:/Idea/AI-Agent/AI-Agent-hm-dianping

# 2. 上传后端
scp -i hmdp-key.pem target/AI-Agent-hm-dianping-0.0.1-SNAPSHOT.jar \
  root@<IP>:/opt/hmdp/

# 3. 上传配置文件
scp -i hmdp-key.pem docker-compose.yml root@<IP>:/opt/hmdp/
scp -i hmdp-key.pem start.sh root@<IP>:/opt/hmdp/
scp -i hmdp-key.pem application-prod.yml root@<IP>:/opt/hmdp/

# 4. 上传前端文件（nginx-1.18.0/html/hmdp 目录）
scp -i hmdp-key.pem -r D:/Idea/nginx-1.18.0/html/hmdp \
  root@<IP>:/opt/hmdp/frontend

# 5. 上传数据库 SQL（如有）
scp -i hmdp-key.pem hmdp.sql root@<IP>:/opt/hmdp/


# ===== 服务器操作 =====
# 6. 安装基础环境
apt update && apt install -y openjdk-17-jdk nginx

# 7. 安装 Docker
curl -fsSL https://get.docker.com | bash

# 8. 修改配置文件中的密码
vi /opt/hmdp/docker-compose.yml
vi /opt/hmdp/application-prod.yml

# 9. 启动依赖服务
cd /opt/hmdp && docker compose up -d

# 10. 初始化数据库（如有 SQL 文件）
docker exec -i mysql mysql -uroot -p<密码> hmdp < /opt/hmdp/hmdp.sql

# 11. 配置 Nginx
cp /etc/nginx/nginx.conf /etc/nginx/nginx.conf.bak
# 将 nginx.conf 替换为以下内容（见第七节）
nginx -t && systemctl reload nginx

# 12. 启动应用
chmod +x /opt/hmdp/start.sh
./start.sh

# 13. 验证
curl http://localhost:8081/shop-type/list
curl http://localhost:80
```

### 阿里云安全组规则

| 端口 | 方向 | 来源 | 用途 |
|------|------|------|------|
| 22 | 入 | 本机 IP | SSH 登录 |
| 80 | 入 | 0.0.0.0/0 | Nginx / 前端入口 |
| 8081 | 入 | 0.0.0.0/0 | 后端直接访问（测试用） |
| 3306 | - | - | 不开放，仅本机访问 |
| 6379 | - | - | 不开放，仅本机访问 |
| 19530 | - | - | 不开放，仅本机访问 |

## 七、Nginx 配置

```nginx
worker_processes  1;

events {
    worker_connections  1024;
}

http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;
    sendfile        on;
    keepalive_timeout  65;

    server {
        listen       80;
        server_name  8.136.136.128;

        # 前端静态文件
        location / {
            root   /opt/hmdp/frontend;
            index  index.html;
            try_files $uri $uri/ /index.html;
        }

        # API 代理到 Spring Boot
        location /api {
            default_type  application/json;
            keepalive_timeout   30s;
            keepalive_requests  1000;
            proxy_http_version 1.1;
            rewrite /api(/.*) $1 break;
            proxy_pass_request_headers on;
            proxy_next_upstream error timeout;
            proxy_pass http://127.0.0.1:8081;
        }
    }
}
```

## 八、开机自启配置

`/etc/systemd/system/hmdp.service`：

```ini
[Unit]
Description=HMDP Application
After=docker.service network-online.target
Requires=docker.service

[Service]
Type=forking
ExecStart=/usr/local/bin/start-hmdp.sh
Restart=no

[Install]
WantedBy=multi-user.target
```

`/usr/local/bin/start-hmdp.sh`：

```bash
#!/bin/bash
cd /opt/hmdp
docker compose -f docker-compose.yml up -d
sleep 10
cd /opt/hmdp && ./start.sh
```

启用自启：

```bash
systemctl daemon-reload
systemctl enable hmdp.service
systemctl enable docker
```

## 十一、已解决的关键问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| MySQL 驱动拒绝连接 | `mysql-connector-java:8.0.30` 的 URL 兼容性 bug | 升级到 `com.mysql:mysql-connector-j:8.0.33` |
| Redis 连接 `192.168.137.128` | `RedissonConfig` 硬编码开发环境 IP | 改用 `@Value` 从 Spring 配置读取 |
| Redis Stream `NOGROUP` 错误 | 新 Redis 缺乏消费组 `g1` | `init()` 中 `XGROUP CREATE ... MKSTREAM` |
| `start.sh` 环境变量不生效 | `nohup` 下 export 和 `--` 参数解析问题 | 改用 `application-prod.yml` + `--spring.profiles.active=prod` |
| Nginx 502 | 安全组未开放 8081 端口 | 阿里云安全组添加 8081 入方向规则 |

## 十二、注意事项

1. **密钥安全**：`application-prod.yml` 中的 DashScope API Key 和数据库密码建议在面试后替换，不要长期暴露
2. **API Key 暴露**：原始 `application.yaml` 中的 `sk-91d6f104...` 已在本文档中保留，建议去 DashScope 控制台重新生成
3. **弹性公网 IP**：停机前务必绑定，否则下次开机 IP 会变，Nginx 配置和 DNS 需同步修改
4. **成本**：按量付费 4C8G 约 300-400 元/月，包年包月约 200-250 元/月。面试后可释放或转包年包月
